#  User Persona

Sistem E-Service dirancang untuk melayani dua tipe pengguna utama dengan kebutuhan yang berbeda.

### 1. Persona Admin (Koordinator)
| Kategori | Keterangan |
| :--- | :--- |
| **Nama** | Ridwan |
| **Pekerjaan** | Admin Operasional E-Service |
| **Background** | Mengelola puluhan pesanan servis setiap hari. Siti sering kesulitan melacak status teknisi dan mencatat laporan keuangan manual. |
| **Goals** | Mendaftarkan orderan dengan cepat, memantau status 'Pending/Selesai' secara real-time, dan membuat laporan pendapatan tanpa error. |
| **Needs** | Antarmuka dashboard yang bersih untuk melihat daftar pengerjaan (`jobs`) dan fitur penugasan teknisi yang mudah. |
| **Pain Points** | Kehilangan jejak catatan pelanggan dan kesalahan penghitungan total pendapatan bulanan. |

### 2. Persona Teknisi ( Lapangan)
| Kategori | Keterangan |
| :--- | :--- |
| **Nama** | Ilham |
| **Pekerjaan** | Teknisi Handal |
| **Background** | Ilham selalu bergerak di lapangan. Ia butuh informasi instruksi kerja yang bisa diakses langsung melalui perangkat mobile. |
| **Goals** | Melihat daftar tugas harian yang terorganisir dan memperbarui status pengerjaan langsung dari lokasi. |
| **Needs** | Akses cepat ke kolom `alamat` dan `deskripsi_masalah` agar tidak perlu sering menelepon admin untuk bertanya. |
| **Pain Points** | Deskripsi kerusakan barang yang kurang jelas dan kesulitan mencatat rincian biaya pengerjaan di lapangan. |
